# Backlog and Future Work

This directory contains planning and prioritization documents for the Purview Security Copilot Lab.

## Contents

- **roadmap-backlog.md** — Comprehensive backlog organized by timeline (Immediate, Short-term, Medium-term, Long-term) with effort estimates, dependencies, and ownership

## Using This Backlog

### For Team Members
- Review items in the **Immediate** section to understand short-term priorities
- Check **Dependencies** before starting work to identify blocking items
- Propose new backlog items as GitHub issues, and they will be triaged into the appropriate timeline

### For Project Managers
- Use **Effort Estimates** to plan sprint capacity and timelines
- Track **Ownership** to ensure clear accountability
- Monitor dependencies to identify critical path items

### For Stakeholders
- **Short-term items** represent Phase 1-3 deliverables (Weeks 1-8)
- **Medium-term items** represent Phase 4-5 deliverables (Weeks 9-15)
- **Long-term items** represent Phase 6 and beyond

## Cadence

The backlog is reviewed and updated:
- **Weekly** during Phase 1-3 (Weeks 1-8)
- **Bi-weekly** during Phase 4-5 (Weeks 9-15)
- **Monthly** during Phase 6 and beyond

## Adding New Items

To propose a new backlog item:

1. Open a GitHub issue with the template "Backlog Item"
2. Include: Title, Description, Effort Estimate, Dependencies, Business Justification
3. Tag for prioritization review
4. Items will be triaged into the appropriate timeline by the project owner

## Key Constraints

- Security Copilot plugin access (required by Week 6)
- Claude Code / Agent SDK access (required by Week 9)
- Purview API stable access (ongoing)
- Team capacity and SME availability
- Alignment with SC and Purview product roadmaps
